# features/detail/

存放复杂 / hybrid / 多 Tab 功能的详细文档。每个文件对应一个功能项，文件名与 `features/_index.md` 中 `detailDoc` 列一致。

仅 `implType` 为 `hybrid` 或复杂度较高的 `frontend-heavy` / `api-driven` 功能才在此建独立文档。

**多 Tab 路由**：`features/_index.md` 的 `detailDoc` 通常指向主文档；子 Tab 可另建文件（如 `competition-scrambles.md` + `competition-group-timer.md`），在 `detailDoc` 或主文档文首用相对链接串联，无需为每个 Tab 单独占一行路由表。
